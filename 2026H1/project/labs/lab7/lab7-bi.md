# Лабораторная работа №7 для ББИ

Спроектировать структуру хранения данных системы с точки зрения бизнес-логики и обеспечения корректности данных.

Ваша задача – определить, какие данные должна хранить система и как они связаны, чтобы:
* избежать ошибок;
* обеспечить целостность данных;
* поддержать бизнес-процессы.

## Ход работы

**По каждом шагу важно дать обоснование согласно прошлым лабораторным работам**

1. Определить данные системы. Ответить:
    * какие данные нужны системе?
    * какие объекты есть в бизнесе?
2. Сформировать структуру данных: сгруппировать данные в сущности, определить основные атрибуты, выделить идентификаторы.
3. Определить связи между сущностями (как объекты связаны между собой, например, `клиент → заказы`, `заказ → товары`).
4. Провести анализ и выделить:
    * где может возникнуть дублирование;
    * где возможны противоречия;
    * как их избежать.
5. Рассмотреть и описать:
    * один объект связан с несколькими (например, заказ и товары);
    * изменение данных во времени (статусы, история).
6. Показать как данные связаны с бизнес-процессами, какие из этих данных критичны.

## Результаты

* Составить схему данных (можно ERD, можно в любом упрощенном виде).
* Описание сущностей: что хранится и зачем.
