# Практический разбор к теме "Сортировка и поиск в массивах"

# Задание №1 (линейный проход)

## Условие

Выведите значение наименьшего положительного числа из списка. Известно, что в списке есть хотя бы один положительный элемент, а значения элементов списка по модулю не превосходят `1000`.

### Формат входных данных

Вводится список чисел через пробел. Все числа находятся на одной строке.

### Формат выходных данных

Одно число (ответ на задачу).

### Тесты

1 input:

```bash
5 -4 3 -2 1
```

1 output:

```bash
1
```

2 input:

```bash
40 -20 10 5 0 2
```

2 output:

```bash
2
```

## Разбор

### Ключевая идея

Нужно найти минимум, но **не среди всех элементов**, а только среди положительных.

### Оптимальный подход

Достаточно одного прохода:

* игнорируем все элементы ≤ 0
* среди оставшихся поддерживаем текущий минимум

### Сложность

* время: `O(n)`
* память: `O(1)`

# Задание №2 (бинарный поиск)

## Условие

В базе данных хранятся идентификаторы пользователей, отсортированные по возрастанию.

Для заданного числа `x` необходимо определить, сколько элементов строго меньше этого значения.

### Формат входных данных

В первой строке вводится число `n` (`1 ≤ n ≤ 100000`).

Во второй строке вводятся `n` целых чисел в отсортированном порядке.

В третьей строке вводится число `x`.

### Формат выходных данных

Выведите одно число – количество элементов, строго меньших `x`.

### Тесты

1 input:

```bash
5
1 3 5 7 9
6
```

1 output:

```bash
3
```

2 input:

```bash
5
1 3 5 7 9
1
```

2 output:

```bash
0
```

## Разбор

### Ключевая идея

Нужно найти **позицию, где начинается `x` или большее значение**.

Это и есть граница: количество элементов слева = ответ.

### Что даёт сортировка

Так как массив отсортирован:

* все элементы < x находятся слева
* все ≥ x – справа

Следовательно, можно искать границу, а не перебирать всё.

### Оптимальный подход

Используем деление диапазона:

* берём середину
* если элемент < x → идём вправо
* иначе → влево

В итоге находим первую позицию, где элемент ≥ x.

### Сложность

* время: `O(log n)`
* память: `O(1)`

# Задание №3 (сортировка)

## Условие

Дан список целых чисел.

Необходимо определить, какое минимальное количество элементов нужно удалить, чтобы оставшаяся последовательность стала неубывающей.

### Формат входных данных

В первой строке вводится число `n` (`1 ≤ n ≤ 100000`).

Во второй строке вводятся `n` целых чисел.

### Формат выходных данных

Выведите одно число – минимальное количество удалений.

### Тесты

1 input:

```bash
5
5 3 4 2 1
```

1 output:

```bash
3
```

2 input:

```bash
6
1 2 3 4 5 6
```

2 output:

```bash
0
```

## Разбор

### Ключевая идея

Удалять минимум элементов = оставить максимум элементов → **найти максимально длинную неубывающую подпоследовательность**.

### Оптимальный подход

Используется структура, которая:

* хранит «лучшие хвосты» последовательностей
* поддерживает их в отсортированном виде

Для каждого элемента:

* ищем место, куда его можно вставить
* обновляем структуру

### Где появляется бинарный поиск

Чтобы быстро найти позицию вставки: используется бинарный поиск по текущим значениям.

### Итог

* длина этой структуры = длина максимальной подпоследовательности
* ответ:

  ```
  n - длина
  ```

### Сложность

* время: `O(n log n)`
* память: `O(n)`
