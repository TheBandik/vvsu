## Задание №1

В аналитической системе обработки данных (например, модуль вычисления формул в Apache Spark) пользователи задают математические выражения в инфиксной форме.

Для выполнения вычислений система должна:

1. преобразовать выражение в постфиксную форму
2. вычислить результат

**Требования:**

* реализовать собственный стек на основе **связного списка**
* использовать классы:
    * `Node`
    * `Stack` (push, pop, top, is_empty)
* реализовать функции:
    * `infix_to_postfix(expression)`
    * `calculate(expression)`
* поддержать:
    * унарный минус (`-3`, `( -2 * 4 )`)
    * оператор степени `**` (правая ассоциативность)
    * вещественные числа (`3.14`)
* обработать ошибки:
    * пустое выражение
    * неизвестные токены
    * несбалансированные скобки

### Формат входных данных

В единственной строке задано выражение `s` (`1 ≤ |s| ≤ 10^5`), где:

* токены разделены пробелами
* допустимы числа, операторы и скобки

### Формат выходных данных

Выведите:

1. постфиксную форму выражения
2. результат вычисления

Если произошла ошибка – вывести сообщение об ошибке.

### Тесты


1 output:



```bash id="pf5"

```


```bash id="pf7"

```


## Задание №2

В backend-сервисе вычислений (например, API на Django или Spring Boot) необходимо реализовать обработку постфиксных выражений с хранением истории вычислений.

Сервис должен поддерживать:

* вычисление выражений
* хранение последних операций
* отмену последнего действия

**Требования:**

Реализовать класс:

```python
class PostfixCalculator
```

Методы:

* `evaluate(expression)` – вычисляет выражение
* `history()` – возвращает список последних операций
* `undo()` – удаляет последнее вычисление

Условия:

* выражение задаётся в постфиксной форме
* токены разделены пробелами
* использовать стек для вычислений
* историю хранить в виде **связного списка ограниченного размера k**

Обработка ошибок:

* деление на ноль
* недостаточно операндов
* лишние операнды

### Формат входных данных

В первой строке задано число `k` (`1 ≤ k ≤ 10^4`) – размер истории.

Во второй строке задано число `n` (`1 ≤ n ≤ 10^5`) – количество выражений.

В следующих `n` строках заданы выражения.

### Формат выходных данных

Выведите:

1. результаты всех вычислений (по одному в строке)
2. историю после всех операций
3. результат после вызова `undo()`

### Тесты

1 input:

```bash id="pc1"

5

5 1 -
```

1 output:

```bash id="pc2"
4

After undo:
[('5 1 -', 4), ('10 2 /', 5)]
```

2 input:

```bash id="pc3"
2
3
2 0 /
3 4 +

```

2 output:

```bash id="pc4"
Error: division by zero

30
```
