# Планировщик процессов

## Проблема планирования в системе

Компьютер должен решать два разных вопроса:
- когда запускать программу;
- как делить CPU между уже запущенными программами.

## Проблема автоматизации задач

Во многих системах существуют задачи, которые должны выполняться регулярно и без участия пользователя.

Примеры задач:
- резервное копирование;
- отправка отчётов;
- обновление системы;
- очистка временных файлов.

Если запускать такие задачи вручную, это:
- неудобно;
- ненадёжно;
- легко забыть.

Поэтому используются **механизмы автоматического планирования задач**.

## Планирование задач и планирование процессов

Важно понимать, что это **разные механизмы**.

### Планирование задач

Когда запустить программу? Например:
```
Каждый день в 03:00
Каждые 5 минут
Раз в неделю
```

В UNIX-системах для этого используется **cron**.

### Планирование процессов

Какой процесс сейчас получит CPU? Это делает **планировщик ядра**.

Связь между ними:
```
cron запускает программу -> 
создаётся процесс -> 
процесс попадает в очередь Ready -> 
планировщик ОС распределяет CPU
```

### Cron

Cron – это системная служба, которая запускает программы по расписанию. Она работает постоянно в фоновом режиме.

#### Таблица cron (crontab)

Все задания cron хранятся в специальных таблицах:
```
crontab
```

Каждый пользователь может иметь собственную таблицу заданий.

Посмотреть задания:
```
crontab -l
```

Редактировать:
```
crontab -e
```

#### Формат записи cron

Каждая строка – это отдельная задача.

Структура:
```
минута час день_месяца месяц день_недели команда
```

Пример записи:
```
0 3 * * * backup.sh
```

Это означает:

```
0 минут
3 часа
каждый день
каждый месяц
любой день недели
запустить backup.sh
```

#### Специальные символы

Cron поддерживает несколько специальных конструкций:
```
# Любое значение

*

# Пример:

* * * * *

# Каждую минуту
```

```
# Шаг

*/

# Пример:

*/5 * * * *

# Каждые 5 минут
```

```
# Список значений

,

# Пример:

1,15,30 * * * *

# Каждые 1, 15 и 30 минут
```

```
# Диапазон значений

-

# Пример:

1-5 * * * *

# Каждые 1-5 минут
```

### Планировщик процессов

Планировщик – это компонент ядра операционной системы, который выбирает процесс для выполнения на CPU.

Основные цели планировщика:
- **справедливость** - каждый процесс должен получать процессорное время;
- **отзывчивость системы** - скорость отклика процесса на действия пользователя должны быть минимальными;
- **пропускная способность** - сколько задач система может выполнить за определённое время;
- **минимизация задержек** - планировщик старается уменьшить время ожидания процессов.

#### Квант времени

Современные ОС используют вытесняющее планирование. Процесс получает CPU на ограниченное время. Это время называется: **квант времени (time slice)**.

После его окончания:
1. ядро прерывает процесс;
2. выполняется переключение контекста;
3. другой процесс получает CPU.

Благодаря этому создаётся эффект одновременной работы программ.

#### Алгоритмы планирования

##### FCFS (First Come First Served)

Процессы выполняются в порядке поступления.

Проблема:длинная задача может задержать все остальные.

##### Round Robin

Каждый процесс получает квант времени.

```
A -> B -> C -> A -> B -> C
```

Это улучшает отзывчивость системы.

##### Приоритетное планирование

Процессы имеют приоритет.

Процессы с высоким приоритетом получают CPU чаще.

##### CFS (Completely Fair Scheduler)

Каждый процесс должен получить справедливую долю CPU.

Планировщик отслеживает:
- сколько времени процесс уже выполнялся;
- выбирает тот процесс, который выполнялся меньше остальных.
