# Лабораторная работа №4

Форма сдачи лабораторной работы:
- архив проекта после проведенного тестирования (команда: `tar -czf lab5.tar.gz data_pipeline`);
- демонстрация таблицы **cron** и работы обработки новых файлов (можно добавить в папку **incoming** файл вручную).

## Задание

Разработать систему автоматической обработки данных с использованием **скриптов** и планировщика задач **cron**.

### Сценарий

Представим, что в системе существует каталог, в который регулярно поступают новые файлы.
Необходимо разработать автоматическую систему, которая:
- генерирует входящие файлы;
- обрабатывает новые файлы;
- ведёт журнал обработки;
- периодически архивирует обработанные данные.

Все операции должны выполняться автоматически через **cron**.

### Шаг 1. Подготовка структуры системы

Создать рабочий каталог:
```
~/data_pipeline
```

Внутри него создать структуру каталогов:
```
incoming        # входящие данные
processed       # обработанные файлы
archive         # архивы
logs            # журналы работы системы
scripts         # исполняемые скрипты
```

### Шаг 2. Генератор входящих файлов

Создать программу, которая будет генерировать тестовые файлы. 

Общие требования:
- файл должен создаваться в каталоге **incoming**; 
- имя файла должно содержать дату и время создания;
- содержимое файла может быть произвольным (например, случайные числа, строки или текущая дата); 
- генератор может быть реализован **bash-скриптом** или **программой на любом языке программирования**.

Скрипт должен:
1. создавать новый файл;
2. записывать в него некоторую информацию;
3. добавлять запись в лог генерации.

Журнал должен храниться в каталоге **logs**.

### Шаг 3. Обработка файлов

Создать скрипт обработки данных.

Скрипт должен:
- проверять каталог **incoming**;
- находить новые файлы;
- перемещать их в каталог **processed**;
- записывать информацию о каждом обработанном файле в журнал.

В журнале должны фиксироваться:
- дата обработки;
- имя файла.

### Шаг 4. Архивирование

Создать скрипт архивирования обработанных данных.

Скрипт должен:
- создавать архив из файлов каталога **processed**;
- сохранять архив в каталог **archive**;
- добавлять запись о выполнении операции в журнал;
- имя архива должно содержать дату и время.

После архивирования обработанные файлы могут быть удалены.

### Шаг 5. Настройка cron

Настроить автоматическое выполнение скриптов.

Необходимо организовать следующую работу системы:
- генерация входящих файлов – каждые 2 минуты;
- обработка новых файлов – каждую минуту;
- архивирование обработанных файлов – каждые 10 минут.

### Шаг 6. Тестирование системы

Дождаться работы генератора и проверить:
- появляются ли новые файлы в каталоге **incoming**;
- перемещаются ли они в каталог **processed**;
- создаются ли архивы;
- ведутся ли журналы работы системы.
