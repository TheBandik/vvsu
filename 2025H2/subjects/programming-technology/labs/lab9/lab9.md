# Лабораторная работа №9

## Задание

Используя произвольные инструменты для генерации тестовых данных создать основные тестовые наборы данных для своего проекта.

## Примеры инструментов

### Универсальные инструменты
- **Mockaroo** — генерация CSV / JSON / SQL по шаблонам
- **JSON Generator / JSON Schema Faker** — генерация данных на основе JSON Schema
- **Seed-скрипты** — собственные генераторы данных, запускаемые при инициализации проекта или БД
- **Property-based testing** — автоматическая генерация входных данных по правилам и ограничениям

### Python
- **Faker** — имена, адреса, телефоны, даты, тексты
- **factory_boy** — генерация объектов для ORM (Django, SQLAlchemy)
- **Fauxfactory** — генерация строк и числовых значений по шаблонам
- **Hypothesis** — property-based генерация входных данных для тестирования логики

### Java
- **Java Faker** — реалистичные тестовые данные
- **Datafaker** — современная альтернатива Java Faker
- **jqwik** — property-based тестирование и генерация данных.

### C#
- **Bogus** — генерация фейковых данных с правилами для моделей
- **AutoFixture** — автоматическая генерация объектов для unit-тестов
- **FsCheck** — property-based генерация тестовых данных

### JavaScript / TypeScript
- **@faker-js/faker** — основной инструмент генерации фейковых данных
- **casual** — простой генератор случайных данных
- **chance** — генерация случайных примитивов и структур
- **fast-check** — property-based генерация данных для тестирования функций
